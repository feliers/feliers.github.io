HADOOP

一、HDFS

介绍

HDFS是一个分布式的文件系统，可用于存储文件。适合一次写入多次读取的模式。但是支持文件修改，如有错误可将错误文件删除后重新上传。

优缺点

  • 优点：

    1. 高容错性。

      数据有多个副本，通过增加副本的形式，提高容错性。

      如果某个副本丢失，其他副本有备份，可进行恢复。

    2. 适合大数据处理

      能够处理大规模的数据(可以是 GB、TB甚至是TB级别的数据)

      能够处理百万规模的文件数量。

    3. 部署成本低

      可以搭建在价格低廉的机器上。

      通过多副本机制，提高可靠性。

  • 缺点

    1. 不适合低延时数据的访问。比如mysql的毫秒级读写数据。

    2. 无法高效的对大量小文件进行存储。

      原因：

      1. 存储小文件会占用namenode 的存储空间，

                2. 小文件存储的寻址时间会超过读取时间。

    3. 不支持并发写入（多个用户写入一个文件）、文件随机修改（更改文件的位置）

二、hdfs常用命令操作

  1. - help

    命令示例：hadoop fs -help rm

  2. - ls 查看目录

    命令示例：hadoop fs -ls /

  3. -mkdir hdfs上创建目录

    hdfs处于安全模式下无法创建目录

    报错信息：mkdir :Cannot create directory /aaa/bbb, Name node is in safe mode

    解决方法：输入命令：hadoop dfsadmin -safemode leave

    提示：Safe mode is OFF

    命令示例：hadoop fs -mkdir -p /aaa/bbb

  4. -moveFromLocal 移动其他目录的文件到hdfs上

    命令示例：hadoop fs -moveFromLocal ./hello.txt /aaa/bbb

  5. -appendToFile 追加一个文件到已存在的文件末尾（将此文件的内容追加到另外一个文件中）

    命令示例：hadoop fs -appendToFile ./word.txt /aaa/bbb/hello.txt

  6. -cat 查看文件内容

    命令示例：hadoop fs -cat /aaa/bbb/hello.txt

  7. -tail 显示文件的末尾

    命令示例：hadoop fs -tail /aaa/bbb/hello.txt

  8. -chown 、-chgrp、-chmod 修改权限

    命令示例：hadoop fs -chown root:root /aaa/bbb/hello.txt

  9. -copyFromLocal 从本地文件系统拷贝文件到HDFS上

    命令示例：hadoop fs -copyFromLocal Readme.md /

  10. -copyToLocal 从HDFS拷贝到本地

    命令示例：hadoop fs -copyToLocal /aaa/bbb/hello.txt ./

  11. -cp 从HDFS的一个路径拷贝到另外一个路径

    命令示例：hadoop fs -cp /aaa/bbb/hello.txt /bigData.txt

  12. -mv 在HDFS中移动文件

    命令示例：hadoop fs -mv /aaa/bbb/hello.txt /aaa/bbb

  13. -rm 删除文件

    4.3、 创建删除节点

    数据节点(znode)。zookeeper数据模型中的数据单元，zookeeper数据模型：

    zookeeper中的四种节点类型：

    1. persistent(持久节点)

      一旦此节点创建成功后，除非主动移除节点，否则此节点会一直保存在zookeeper上。

    2. ephemeral(临时节点)

      此节点的生命周期是和客户端的会话相关连的，一旦会话结束，该会话上的临时节点都会被自动移除。

    3. persistent-sequential(持久有序节点)

    4. ephemeral-sequential(持久有序节点)

​ 创建/删除节点流程

  1. 启动服务器，并使用zkCli.sh连接服务器，进入客户端命令行界面。

    zkServer.sh start

    zkCli.sh -server 127.0.0.1:2181

  2. 进入zookeeper客户端后，可通过命令“ls /” 查看

  3. 创建节点：使用以下命令创建

    create

    创建名称为/zoo1的节点。（持久节点）

    创建/zoo1,/zoo2,/zoo3是后面的空字符串（“”）表示当前节点不保存数据。如果想将节点设置为可以保存任何字符串，则将“”替换为workers.注：在创建节点时，没有数据也要指定一个空字符串”“

    eg：create /zoo1 “”

    创建临时节点（参数：-e）

    eg: create -s /zoo2 “”

    创建有序节点（参数：-s）

    eg: create -s /zoo3 “”

  4. 删除节点

    删除节点

    delete

    删除/zoo1节点

    eg:delete /zoo1

    ​ false

    修改regionServers文件

    vi regionServers

    master

    slave1

    slave2

    slave3

    配置环境变量

    vi /etc/profile

    export HBASE_HOME=……

    export PATH=$PATH: $HBASE_HOME/bin

  3. 将配置好的hbase分发到其他机器上

    scp -r $HBASE_HOME @slave2:/……

  4. 验证是否配置成功

    启动hbase

    start-dfs.sh

    zkServer.sh start

    start-hbase.sh

1、读取文件

  1. 使用FileSystem对象打开输入流，从中获取数据。

flume

安装配置

  1. 解压flume压缩文件

    tar -zxvf apach-flume-1.9.0-bin.tar.gz

  2. 进入flume/conf下新建flume.conf文件，并输入以下内容

    //定义这个agent中各组件的名字

    a1.sources = r1

    a1.sinks = k1

    a1.channels = c1

    //描述和配置source组件：r1

    a1.sources.r1.type = netcat

    a1.sources.r1.bind = localhost <–

    a1.sources.r1.port = 44444

    //描述和配置sink组件：k1

    a1.sinks.k1.type = logger

    // 描述和配置channel组件，此处使用是内存缓存的方式

    a1.channels.c1.type = memory

    a1.channels.c1.capacity = 1000

    a1.channels.c1.transactionCapacity = 100

    //描述和配置source channel sink之间的连接关系

    a1.sources.r1.channels = c1

    a1.sinks.k1.channel = c1

  3. 修改 flume-env.sh 文件

    在 flume/conf 目录下执行：cp flume-env.sh.tamplate flume-env.sh

    vi flume-env.sh

    增加：export JAVA_HOME=……

  4. 配置环境变量(vi /etc/profile)

    输入以下内容

    export FLUME_HOME=/

    export PATH=$PATH:$FLUME_HOME/bin

  5. 启动agent

    flume-ng agent -c conf -f /flume/conf/flume.conf -n a1 -Dflume.root.logger=INFO,console

  6. 测试

    在另外一台可以和agent节点联网的机器上输入以下内容

    telnet ip port

    eg : telnet 192.168.43.11 10086
